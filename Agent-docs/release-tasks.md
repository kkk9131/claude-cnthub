# Release Tasks

## リリース前（必須）
- 既知バグの修正と再発防止の確認
- 必須機能の仕上げ（未完のTODOを消し込み）
- クリティカルな回帰の確認（主要フローの手動スモーク）
- テスト実行と失敗の解消（`bun test`）
- ビルド検証（`bun run build`）
- リリースノートに反映する変更点の整理

## ポストリリース（改善）
- 大きめのリファクタ（責務分離・分割）
- 構造整理（ディレクトリ/モジュールの整理）
- 技術的負債の解消（重複ロジック/巨大ファイル/共通化）
- 型の厳密化（`any` や `@ts-nocheck` の削減）
- ログ/エラー処理の統一（logger 集約・出力制御）
- テストの共通化と高速化（共通ヘルパー整備）
